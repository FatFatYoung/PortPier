## 数据流

### 1. 客户端连接流程

```
客户端                    服务端
   │                        │
   │  1. TCP 连接            │
   │ ────────────────────────>│
   │                        │
   │  2. 发送 client_id      │
   │ ────────────────────────>│
   │                        │
   │  3. 发送 token          │
   │ ────────────────────────>│
   │                        │
   │  4. 认证结果            │
   │ <────────────────────────│
   │                        │
   │  5. 发送规则            │
   │ ────────────────────────>│
   │                        │
   │  6. 心跳保活            │
   │ <───────────────────────>│
```

### 2. 数据转发流程

```
访客                    服务端                    客户端                    本地服务
 │                        │                        │                        │
 │  1. HTTP 请求           │                        │                        │
 │ ────────────────────────>│                        │                        │
 │                        │                        │                        │
 │  2. 分配 stream_id      │                        │                        │
 │                        │ ────────────────────────>│                        │
 │                        │                        │                        │
 │  3. 转发数据            │                        │                        │
 │ ────────────────────────>│ ────────────────────────>│ ────────────────────────>│
 │                        │                        │                        │
 │  4. 返回响应            │                        │                        │
 │ <────────────────────────│ <────────────────────────│ <────────────────────────│
```

## 核心组件

### 服务端组件

1. **GUI Server** (`gui_server.py`)
   - tkinter 可视化管理界面
   - 客户端管理
   - 规则配置
   - 日志查看
   - 防火墙日志
   - IP 访问控制

2. **Protocol Handler** (`common.py`)
   - 数据包编码/解码
   - 流量控制
   - 心跳机制

3. **Client Handler**
   - 客户端连接管理
   - 认证验证
   - 规则同步
   - 数据转发

### 客户端组件

1. **GUI Client** (`gui_client.py`)
   - tkinter 可视化配置界面
   - 服务器配置
   - 规则管理
   - 日志查看

2. **Proxy Client**
   - 本地端口监听
   - 数据转发
   - 自动重连

## 协议格式

### 数据包格式

```
+──────────────+──────────────+──────────────+
│ Length (4B)  │ stream_id    │ Payload      │
│              │ (4B)         │ (变长)       │
+──────────────+──────────────+──────────────+
```

### stream_id 含义

| stream_id | 含义 |
|-----------|------|
| 0 | 控制消息（心跳、规则操作） |
| > 0 | 数据流（对应具体的连接） |

### 控制消息格式

```json
{
  "action": "add|remove|pong",
  "public_port": 80,
  "target_host": "127.0.0.1",
  "target_port": 8080
}
```

## 安全机制

### 1. 认证机制

- 客户端连接时发送 client_id 和 token
- 服务端验证 token 是否匹配
- token 使用 PBKDF2 算法加密存储

### 2. IP 访问控制

- 客户端连接 IP 白名单/黑名单
- 访客访问 IP 白名单/黑名单
- 支持 CIDR、IP 范围、通配符格式

### 3. 密码保护

- 客户端 GUI 需要密码登录
- 密码使用 PBKDF2 算法加密存储
- 支持修改密码

## 性能优化

### 1. 异步架构

- 使用 Python asyncio 实现异步 I/O
- 支持高并发连接

### 2. 缓冲区优化

- 读取缓冲区：64KB
- 写入缓冲区：64KB
- 减少系统调用次数

### 3. TCP Keepalive

- 开启 TCP Keepalive 防止连接超时
- 参数：60s 空闲，15s 间隔，5 次重试

## 扩展性

### 1. 添加新的传输协议

在 `common.py` 中扩展编码/解码函数。

### 2. 添加新的认证方式

在服务端 `handle_client` 中修改认证逻辑。

### 3. 添加新的访问控制

在服务端添加新的检查函数，参考 `is_ip_allowed`。
